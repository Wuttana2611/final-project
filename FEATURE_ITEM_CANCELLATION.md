# ฟีเจอร์การยกเลิกรายการอาหาร (Item Cancellation Feature)

## ภาพรวม
ฟีเจอร์นี้ช่วยให้ลูกค้าสามารถยกเลิกรายการอาหารบางรายการได้ แต่จะไม่สามารถยกเลิกได้หากพ่อครัวเริ่มปรุงแล้ว

## การทำงาน

### 1. เงื่อนไขการยกเลิก
- ✅ **สามารถยกเลิกได้**: เมื่อออเดอร์อยู่ในสถานะ "รอดำเนินการ" (pending) 
- ❌ **ไม่สามารถยกเลิกได้**: เมื่อพ่อครัวกดเริ่มปรุงแล้ว (สถานะ preparing, ready, served)

### 2. หน้าจอที่เกี่ยวข้อง

#### 📱 หน้าเช็คบิล (customer/bill.php)
- แสดงรายการอาหารพร้อมปุ่มยกเลิก (ถ้าสามารถยกเลิกได้)
- แสดง Modal สำหรับเลือกจำนวนที่ต้องการยกเลิก
- อัปเดตข้อมูลแบบ real-time หลังยกเลิก

#### 👨‍🍳 หน้าครัว (chef/kitchen.php)  
- เมื่อกดเริ่มปรุง จะไม่สามารถยกเลิกรายการในออเดอร์นั้นได้อีก
- อัปเดตฟิลด์ `can_cancel = FALSE` สำหรับรายการทั้งหมดในออเดอร์

### 3. การเปลี่ยนแปลงฐานข้อมูล

#### ตาราง `order_items` (เพิ่มฟิลด์ใหม่):
```sql
cancelled_quantity INT DEFAULT 0      -- จำนวนที่ยกเลิก
cancelled_at TIMESTAMP NULL           -- เวลาที่ยกเลิก  
can_cancel BOOLEAN DEFAULT TRUE       -- สามารถยกเลิกได้หรือไม่
```

### 4. API Endpoints

#### 🔗 `/api/cancel-item.php` (POST)
การยกเลิกรายการอาหาร

**Request Body:**
```json
{
    "order_item_id": 123,
    "cancel_quantity": 1,
    "session_token": "abc123"
}
```

**Response:**
```json
{
    "success": true,
    "message": "ยกเลิกรายการสำเร็จ 1 รายการ",
    "new_total": "150.00",
    "cancelled_quantity": 1,
    "remaining_quantity": 2
}
```

**การตรวจสอบ:**
- ✅ ตรวจสอบ session token
- ✅ ตรวจสอบสถานะออเดอร์
- ✅ ตรวจสอบฟิลด์ `can_cancel`
- ✅ ตรวจสอบจำนวนที่เหลือ

### 5. UI/UX Features

#### 🎨 การแสดงผล
- **รายการปกติ**: แสดงจำนวนเต็มพร้อมปุ่มยกเลิก
- **รายการที่ยกเลิกบางส่วน**: แสดงจำนวนที่เหลือ + ข้อมูลการยกเลิก
- **รายการที่ไม่สามารถยกเลิก**: ซ่อนปุ่มยกเลิก + แสดงข้อความ "ไม่สามารถยกเลิก"

#### 📱 Modal การยกเลิก
- เลือกจำนวนที่ต้องการยกเลิก (ปุ่ม +/-)
- แสดงคำเตือนเรื่องการยกเลิกที่ไม่สามารถย้อนกลับ
- ยืนยันการยกเลิกพร้อมการแจ้งเตือน

### 6. การติดตั้งและการใช้งาน

#### 🔧 ขั้นตอนการติดตั้ง
1. รันไฟล์ `migration_add_cancel_functionality.sql`
```sql
mysql -u root -p restaurant_qrcode < migration_add_cancel_functionality.sql
```

2. อัปเดตไฟล์ที่เกี่ยวข้อง:
   - ✅ `customer/bill.php` - เพิ่ม UI การยกเลิก
   - ✅ `chef/kitchen.php` - อัปเดตการปิดการยกเลิก
   - ✅ `api/cancel-item.php` - API ใหม่สำหรับยกเลิก

#### 🚀 วิธีการใช้งาน
1. **ลูกค้า**: เข้าหน้าเช็คบิล → คลิกปุ่ม ❌ → เลือกจำนวน → ยืนยัน
2. **พ่อครัว**: เมื่อกด "เริ่มปรุง" ลูกค้าจะไม่สามารถยกเลิกได้อีก

### 7. การทดสอบ

#### ✅ Test Cases
1. **ยกเลิกรายการเมื่อ pending** → ควรสำเร็จ
2. **ยกเลิกรายการเมื่อ preparing** → ควรไม่สำเร็จ  
3. **ยกเลิกจำนวนที่มากกว่าที่มี** → ควรไม่สำเร็จ
4. **ยกเลิกรายการทั้งหมด** → ออเดอร์ควรเป็น cancelled
5. **อัปเดตยอดรวม** → ควรคำนวณใหม่ถูกต้อง

### 8. ประโยชน์
- 🎯 **ประสบการณ์ลูกค้าดีขึ้น**: ความยืดหยุ่นในการเปลี่ยนแปลงออเดอร์
- 🍳 **ลดความสูญเสีย**: ป้องกันการปรุงอาหารที่ลูกค้าไม่ต้องการ
- 💰 **การเงินแม่นยำ**: คำนวณยอดใหม่อัตโนมัติ
- ⏰ **ประหยัดเวลา**: ลดการสื่อสารกับพนักงาน
